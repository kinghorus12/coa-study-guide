Day 1 RuleComplete all tasks from scratch in under 15 minutes total with zero lookups by end of day.
Task 1.1 — Exam Style
Verify your OpenStack environment is fully healthy. List all registered services, endpoints, and confirm your current user context.
# Verify services registered
openstack service list

# Verify all endpoints exist (public, internal, admin)
openstack endpoint list

# Show service catalog
openstack catalog list

# Confirm your context
openstack token issue -c username -c project_name -c expires
Task 1.2 — Exam Style
Create a project called dev-team. Create user alice with password SecurePass123. Assign alice the member role on dev-team. Verify the assignment.
# Create project in Default domain
openstack project create dev-team

# Create user
openstack user create alice --password SecurePass123

# Assign role
openstack role add --project dev-team --user alice member

# VERIFY — always do this
openstack role assignment list --project dev-team --user alice --names
Task 1.3 — Exam Style
Create 3 users: bob, carol, dave. Create group developers. Add all 3 to the group. Assign developers the reader role on project demo.
for user in bob carol dave; do
  openstack user create $user --password Pass123
done

openstack group create developers

for user in bob carol dave; do
  openstack group add user developers $user
done

openstack role add --project demo --group developers reader

# Verify group membership
openstack group contains user developers bob
openstack role assignment list --project demo --names
Task 1.4 — Exam Style (Custom Domain)
Create domain acme. Inside it, create project acme-prod and user acme-admin. Assign acme-admin the admin role on acme-prod.
# Create domain
openstack domain create acme

# Create project in domain
openstack project create acme-prod --domain acme

# Create user in domain
openstack user create acme-admin --domain acme --password AcmePass123

# Assign role — note both --user-domain and --project-domain flags
openstack role add \
  --project acme-prod --project-domain acme \
  --user acme-admin --user-domain acme \
  admin

# Verify
openstack role assignment list --project acme-prod --project-domain acme --names
🎯
Day 1 Goal: All 4 tasks from scratch in 15 minutes. No notes.

Identity · 2–3 hrs

Task 2.1 — Exam Style
Project dev-team is consuming too many resources. Set its quota to: max 5 instances, 10 vCPUs, 20480 MB RAM, 10 volumes, 100 GB total volume storage.
# Set compute quota
openstack quota set dev-team \
  --instances 5 \
  --cores 10 \
  --ram 20480

# Set volume quota
openstack quota set dev-team \
  --volumes 10 \
  --gigabytes 100

# Verify everything
openstack quota show dev-team
Task 2.2 — Exam Style
Show the current quota usage for project dev-team — both limits and current consumption.
# Show quota with usage data
openstack quota show --usage dev-team

# Show just compute usage
openstack limits show --absolute

Open docs.openstack.org. Find the CLI reference for openstack role add. Time yourself — target: under 45 seconds. Then find the openstack quota set options.
Bookmark path: docs.openstack.org → python-openstackclient → CLI Reference → openstack role
Task 2.4 — Exam Style
Disable project dev-team so users can no longer log in. Then re-enable it.

openstack project set dev-team --disable

openstack project show dev-team -c enabled

Glance — Image Management
Images 5% · 2 hrs · Do early on exam day
▾
Task 3.1 — Exam Style
Upload file ubuntu-22.04.qcow2 as image ubuntu-jammy. Set format qcow2, make it public, add property os_type=linux and tag production.
openstack image create ubuntu-jammy \
  --disk-format qcow2 \
  --container-format bare \
  --file ubuntu-22.04.qcow2 \
  --public \
  --property os_type=linux \
  --tag production

# Verify
openstack image show ubuntu-jammy -c visibility -c status -c properties
Task 3.2 — Exam Style
Change image ubuntu-jammy visibility to private. Then deactivate it. Then reactivate it.
# Change visibility
openstack image set ubuntu-jammy --private

# Deactivate
openstack image deactivate ubuntu-jammy

# Verify status = deactivated
openstack image show ubuntu-jammy -c status -c visibility

# Reactivate
openstack image reactivate ubuntu-jammy

# Make public again for use in compute tasks
openstack image set ubuntu-jammy --public
Task 3.3 — Exam Style
Share image ubuntu-jammy with project dev-team. Then accept the image from the dev-team project context.
# Get project ID
PROJECT_ID=$(openstack project show dev-team -f value -c id)

# Share image
openstack image add project ubuntu-jammy $PROJECT_ID
openstack image set ubuntu-jammy --shared

# To accept (switch to dev-team context first)
openstack --os-project-name dev-team image set ubuntu-jammy --accept

# Verify shared members
openstack image member list ubuntu-jammy
🎯
Day 3 Goal: Full image lifecycle in under 10 minutes. Images are 5% of marks — fast points.

Nova Part 1 — Launch, Flavors, Keypairs, Console
Compute 35% · 3 hrs · Biggest mark pool
▾
Task 4.1 — Exam Style
Create flavor app.small with ID 101: 2 vCPUs, 2048 MB RAM, 20 GB disk. Create flavor app.micro with 1 vCPU, 512 MB RAM, 10 GB disk.
openstack flavor create app.small \
  --id 101 --vcpus 2 --ram 2048 --disk 20

openstack flavor create app.micro \
  --vcpus 1 --ram 512 --disk 10

# Verify
openstack flavor list
Task 4.2 — Exam Style
Create keypair exam-key. Then import an existing public key from ~/.ssh/id_rsa.pub as keypair imported-key.
# Generate new keypair (saves private key)
openstack keypair create exam-key > ~/exam-key.pem
chmod 600 ~/exam-key.pem

# Import existing public key
openstack keypair create imported-key \
  --public-key ~/.ssh/id_rsa.pub

# List keypairs
openstack keypair list
Task 4.3 — Exam Style
Create security group web-sg. Add rules: allow SSH (22) from anywhere, HTTP (80) from anywhere, HTTPS (443) from anywhere, ICMP from anywhere. Launch instance web-01 using ubuntu-jammy, app.small flavor, private network, exam-key keypair, and web-sg security group.
# Security group
openstack security group create web-sg --description "Web servers"

openstack security group rule create web-sg --protocol tcp --dst-port 22 --remote-ip 0.0.0.0/0
openstack security group rule create web-sg --protocol tcp --dst-port 80 --remote-ip 0.0.0.0/0
openstack security group rule create web-sg --protocol tcp --dst-port 443 --remote-ip 0.0.0.0/0
openstack security group rule create web-sg --protocol icmp --remote-ip 0.0.0.0/0

# Launch instance
openstack server create web-01 \
  --image ubuntu-jammy \
  --flavor app.small \
  --network private \
  --key-name exam-key \
  --security-group web-sg \
  --wait

# Verify ACTIVE
openstack server show web-01 -c status -c addresses -c security_groups
Task 4.4 — Exam Style
Get the VNC console URL for web-01. Show the last 30 lines of its console log. Show its current power state.
openstack console url show web-01 --novnc
openstack console log show web-01 --lines 30
openstack server show web-01 -c power_state -c status -c vm_state
🎯
Day 4 Goal: Launch 5 instances with different configs. Under 3 minutes each. No notes.

Nova Part 2 — Snapshots, Resize, Rebuild, Lifecycle
Compute 35% · 3 hrs
▾
Task 5.1 — Exam Style
Snapshot running instance web-01 as web-01-snap. Launch new instance web-02 from that snapshot. Verify both are ACTIVE.
# Take snapshot (instance stays running)
openstack server image create web-01 --name web-01-snap --wait

# Verify snapshot is ACTIVE
openstack image show web-01-snap -c status

# Launch from snapshot
openstack server create web-02 \
  --image web-01-snap \
  --flavor app.small \
  --network private \
  --wait

openstack server list -c Name -c Status
Task 5.2 — Exam Style
Resize web-01 to flavor app.micro. Confirm the resize. Verify the new flavor is applied.
⚠️
Must Confirm!After resize, instance is in VERIFY_RESIZE state. It stays there forever unless you run confirm or revert. This trips many candidates.
openstack server resize web-01 --flavor app.micro --wait

# Status = VERIFY_RESIZE — must confirm
openstack server show web-01 -c status

# Confirm the resize
openstack server resize confirm web-01

# Verify new flavor
openstack server show web-01 -c flavor
Task 5.3 — Exam Style
Rebuild web-01 using image ubuntu-jammy. Set admin password to RebuildPass456. Verify it returns to ACTIVE.
openstack server rebuild web-01 \
  --image ubuntu-jammy \
  --admin-password RebuildPass456 \
  --wait

openstack server show web-01 -c status -c image
Task 5.4 — Exam Style
Demonstrate all power state operations on web-01: stop, start, soft-reboot, hard-reboot, suspend, resume, pause, unpause.
openstack server stop web-01
openstack server show web-01 -c status          # SHUTOFF
openstack server start web-01
openstack server reboot web-01                  # soft reboot
openstack server reboot web-01 --hard           # hard reboot
openstack server suspend web-01
openstack server resume web-01
openstack server pause web-01
openstack server unpause web-01
🎯
Day 5 Goal: Snap → launch → resize → confirm → rebuild → power states. All from memory.

Nova Part 3 — Server Groups, Host Aggregates, User Data
Compute 35% · Extra depth · 2 hrs
▾
Task 6.1 — Exam Style
Create server group web-antiaffinity with anti-affinity policy. Launch two instances in this group so they land on different hypervisors.
openstack server group create web-antiaffinity --policy anti-affinity

GROUP_ID=$(openstack server group show web-antiaffinity -f value -c id)

openstack server create web-aa-1 \
  --image ubuntu-jammy --flavor app.small \
  --network private --hint group=$GROUP_ID --wait

openstack server create web-aa-2 \
  --image ubuntu-jammy --flavor app.small \
  --network private --hint group=$GROUP_ID --wait

# Verify they are on different hosts
openstack server show web-aa-1 -c "OS-EXT-SRV-ATTR:host"
openstack server show web-aa-2 -c "OS-EXT-SRV-ATTR:host"
Task 6.2 — Exam Style
Create host aggregate ssd-hosts with metadata storage=ssd. Create availability zone az-ssd from it. Launch instance ssd-vm in that availability zone.
openstack aggregate create ssd-hosts
openstack aggregate set --property storage=ssd ssd-hosts
openstack aggregate set --zone az-ssd ssd-hosts

openstack server create ssd-vm \
  --image ubuntu-jammy --flavor app.small \
  --network private \
  --availability-zone az-ssd \
  --wait
Task 6.3 — Exam Style
Launch instance userdata-vm that automatically installs nginx on first boot using cloud-init user data.
cat > /tmp/init.sh << 'EOF'
#!/bin/bash
apt-get update -y
apt-get install -y nginx
systemctl enable nginx
systemctl start nginx
EOF

openstack server create userdata-vm \
  --image ubuntu-jammy \
  --flavor app.small \
  --network private \
  --user-data /tmp/init.sh \
  --wait

# Check console log to verify cloud-init ran
openstack console log show userdata-vm --lines 50
🎯
Day 6 Goal: Server groups, availability zones, and user-data from memory.
