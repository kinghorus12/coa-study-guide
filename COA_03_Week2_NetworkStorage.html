Neutron Part 1 — Networks & Subnets
Networking 30% · Provider + Tenant networks · 3 hrs
▾
Task 8.1 — Exam Style
Create tenant network app-net. Create subnet app-subnet on it: CIDR 192.168.100.0/24, gateway 192.168.100.1, DNS 8.8.8.8 and 8.8.4.4, DHCP enabled.
openstack network create app-net

openstack subnet create app-subnet \
  --network app-net \
  --subnet-range 192.168.100.0/24 \
  --gateway 192.168.100.1 \
  --dns-nameserver 8.8.8.8 \
  --dns-nameserver 8.8.4.4

# Verify
openstack subnet show app-subnet
Task 8.2 — Exam Style
Create external provider network public using flat type on physical network extnet. Create subnet public-subnet: CIDR 203.0.113.0/24, gateway 203.0.113.1, allocation pool 203.0.113.100–203.0.113.200, no DHCP.
# Admin only — external network
openstack network create public \
  --external \
  --provider-network-type flat \
  --provider-physical-network extnet

openstack subnet create public-subnet \
  --network public \
  --subnet-range 203.0.113.0/24 \
  --gateway 203.0.113.1 \
  --allocation-pool start=203.0.113.100,end=203.0.113.200 \
  --no-dhcp

openstack network show public -c "router:external"
Task 8.3 — Exam Style
Create a second network db-net with subnet db-subnet: CIDR 10.20.0.0/24, DHCP disabled.
openstack network create db-net
openstack subnet create db-subnet \
  --network db-net \
  --subnet-range 10.20.0.0/24 \
  --no-dhcp

openstack network list
🎯
Day 8 Goal: Create any network + subnet configuration in under 5 minutes.

Neutron Part 2 — Routers & Floating IPs
Networking 30% · The most common exam scenario · 3 hrs
▾
💡
You Have Done This AlreadyMicroStack floating IP setup is exactly this workflow. Run it from memory first, then compare with the solution below.
Task 9.1 — Exam Style
Create router main-router. Set its external gateway to public network. Connect it to app-subnet and db-subnet.
openstack router create main-router

# Set external gateway
openstack router set main-router --external-gateway public

# Add internal interfaces
openstack router add subnet main-router app-subnet
openstack router add subnet main-router db-subnet

# Verify
openstack router show main-router -c external_gateway_info
openstack router port list main-router
Task 9.2 — Exam Style
Allocate a floating IP from public network. Associate it to instance web-01. Verify the instance is reachable externally.
openstack floating ip create public

# Get the allocated IP
FIP=$(openstack floating ip list -f value -c "Floating IP Address" | head -1)

# Associate to instance
openstack server add floating ip web-01 $FIP

# Verify
openstack server show web-01 -c addresses
openstack floating ip show $FIP -c status -c fixed_ip_address
Task 9.3 — Speed Challenge
From zero: create network → subnet → router with gateway → launch instance → floating IP → associate. Target: under 12 minutes total.
# Full end-to-end in one session — time yourself
openstack network create speed-net
openstack subnet create speed-sub --network speed-net --subnet-range 10.99.0.0/24 --dns-nameserver 8.8.8.8
openstack router create speed-router
openstack router set speed-router --external-gateway public
openstack router add subnet speed-router speed-sub
openstack server create speed-vm --image ubuntu-jammy --flavor app.small --network speed-net --security-group web-sg --wait
FIP=$(openstack floating ip create public -f value -c floating_ip_address)
openstack server add floating ip speed-vm $FIP
openstack server show speed-vm -c addresses

Neutron Part 3 — Ports, Troubleshooting, Advanced
Networking 30% · Diagnostics · 2 hrs
▾
Task 10.1 — Exam Style
Create port static-port on app-net with fixed IP 192.168.100.50. Launch instance static-vm using that specific port.
openstack port create static-port \
  --network app-net \
  --fixed-ip subnet=app-subnet,ip-address=192.168.100.50

openstack server create static-vm \
  --image ubuntu-jammy \
  --flavor app.small \
  --port static-port \
  --wait

openstack server show static-vm -c addresses
Task 10.2 — Exam Style
Instance web-01 has no internet access. Diagnose the problem using CLI only. List your complete diagnostic steps.
# Step 1: Check instance status and IPs
openstack server show web-01 -c status -c addresses -c security_groups

# Step 2: Is floating IP associated?
openstack floating ip list

# Step 3: Check security group allows outbound
openstack security group rule list web-sg

# Step 4: Check router has external gateway
openstack router show main-router -c external_gateway_info

# Step 5: Check router has subnet interface
openstack router port list main-router

# Step 6: Check network is admin state UP
openstack network show app-net -c admin_state_up -c status

# Step 7: Check ports on instance
openstack port list --server web-01
Task 10.3 — Exam Style
Allow port static-port to accept traffic for VIP address 192.168.100.100 (allowed address pair).
PORT_ID=$(openstack port show static-port -f value -c id)

openstack port set $PORT_ID \
  --allowed-address ip-address=192.168.100.100

openstack port show static-port -c allowed_address_pairs
🎯
Day 10 Goal: Diagnose any broken network path in under 6 minutes using only CLI.

Cinder — Block Storage
Block Storage 10% · Volumes · Snapshots · BFV · 2 hrs
▾
Task 11.1 — Exam Style
Create 20 GB volume data-vol. Attach it to web-01. Detach it. Extend it to 30 GB. Take a snapshot called data-snap. Create new volume data-restored from snapshot.
openstack volume create data-vol --size 20

# Wait for available
openstack volume show data-vol -c status

openstack server add volume web-01 data-vol
openstack volume show data-vol -c attachments

openstack server remove volume web-01 data-vol
openstack volume set data-vol --size 30
openstack volume show data-vol -c size

# Snapshot (use --force if still in-use)
openstack volume snapshot create data-snap --volume data-vol

# Restore from snapshot
openstack volume create data-restored --snapshot data-snap --size 30
openstack volume show data-restored -c status
Task 11.2 — Exam Style
Launch instance bfv-vm that boots directly from a new 20 GB volume based on image ubuntu-jammy. Verify the volume is attached as boot disk.
openstack server create bfv-vm \
  --image ubuntu-jammy \
  --flavor app.small \
  --network app-net \
  --boot-from-volume 20 \
  --wait

openstack server show bfv-vm -c volumes_attached
🎯
Day 11 Goal: Full volume lifecycle (create, attach, detach, extend, snapshot, restore) in under 12 minutes.

Swift — Object Storage
Object Storage 5% · ACLs · Metadata · 2 hrs
▾
⚠️
Memorise Swift ACL SyntaxSwift ACL strings are non-obvious. Candidates who haven't drilled them specifically lose these marks. Study the patterns below until they are automatic.
Task 12.1 — Exam Style
Create container backups. Upload local file /etc/hostname to it. Download it as /tmp/hostname-copy.
openstack container create backups

openstack object create backups /etc/hostname
# Note: object name will be "etc/hostname" — rename with --name
openstack object create backups /etc/hostname --name hostname.txt

openstack object list backups

openstack object save backups hostname.txt --file /tmp/hostname-copy
cat /tmp/hostname-copy
Task 12.2 — Exam Style
Make container backups publicly readable by anyone. Verify. Then make it private again.
# Make public — ACL string is EXACT
openstack container set backups \
  --property "X-Container-Read=.r:*,.rlistings"

# Verify ACL is set
openstack container show backups

# Make private (remove the ACL)
openstack container unset backups --property "X-Container-Read"

openstack container show backups
Task 12.3 — Exam Style
Set object hostname.txt in container backups to auto-delete after 3600 seconds.
openstack object set backups hostname.txt \
  --property "X-Delete-After=3600"

openstack object show backups hostname.txt
Swift ACL Cheatsheet — Memorise All Patterns
# Public read (anyone)
--property "X-Container-Read=.r:*,.rlistings"

# Specific project can read
--property "X-Container-Read=PROJECT_ID:*"

# Specific user in project can read
--property "X-Container-Read=PROJECT_ID:USER_ID"

# Project can write
--property "X-Container-Write=PROJECT_ID:*"

# Remove ACL
openstack container unset CONTAINER --property "X-Container-Read"

# Auto-delete object after N seconds
--property "X-Delete-After=N"

# Auto-delete at specific Unix timestamp
--property "X-Delete-At=1735689600"
🎯
Day 12 Goal: All Swift ACL patterns from memory. No hesitation on X-Container-Read syntax.

Heat — Orchestration & YAML Templates
Orchestration ~10% · Full template examples · 3 hrs
▾
HOT Template Structure — 5 Sections to Memorise
# heat_template_version — always use 2021-04-16
heat_template_version: 2021-04-16

description: What this stack does

# parameters — optional input values
parameters:
  param_name:
    type: string        # string, number, boolean, json, comma_delimited_list
    description: What it does
    default: default_value
    constraints:
      - allowed_values: [val1, val2]

# resources — REQUIRED — actual cloud resources
resources:
  resource_logical_name:
    type: OS::ServiceName::ResourceType
    properties:
      property_name: value

# outputs — optional return values
outputs:
  output_name:
    description: What this returns
    value: { get_attr: [resource_name, attribute] }
Intrinsic Functions — Know All 4
# Reference another resource in this stack
{ get_resource: my_network }

# Get an input parameter
{ get_param: flavor_name }

# Get attribute from deployed resource
{ get_attr: [server, first_address] }
{ get_attr: [server, addresses, network-name, 0, addr] }
{ get_attr: [fip, floating_ip_address] }
{ get_attr: [network, id] }

# String replace
{ str_replace:
    template: "Hello $NAME, welcome to $PLACE"
    params:
      $NAME: { get_param: username }
      $PLACE: OpenStack }
Task 13.1 — Exam Style: Write and Deploy
Write a Heat template that creates: 1 network, 1 subnet (10.10.0.0/24), 1 instance. Accept image and flavor as parameters. Output the instance IP.
# File: /tmp/basic-stack.yaml
heat_template_version: 2021-04-16
description: Basic stack with network and instance

parameters:
  image_name:
    type: string
    default: ubuntu-jammy
  flavor_name:
    type: string
    default: app.small

resources:

  stack_network:
    type: OS::Neutron::Net
    properties:
      name: stack-net

  stack_subnet:
    type: OS::Neutron::Subnet
    properties:
      network: { get_resource: stack_network }
      cidr: 10.10.0.0/24
      dns_nameservers: [8.8.8.8]
      enable_dhcp: true

  stack_instance:
    type: OS::Nova::Server
    properties:
      name: stack-vm
      image: { get_param: image_name }
      flavor: { get_param: flavor_name }
      networks:
        - network: { get_resource: stack_network }

outputs:
  vm_ip:
    description: Instance IP address
    value: { get_attr: [stack_instance, first_address] }
  network_id:
    description: Network ID
    value: { get_attr: [stack_network, id] }
# Validate template BEFORE deploying
openstack orchestration template validate -t /tmp/basic-stack.yaml

# Deploy
openstack stack create mystack -t /tmp/basic-stack.yaml --wait

# Deploy with parameter overrides
openstack stack create mystack2 -t /tmp/basic-stack.yaml \
  --parameter image_name=cirros \
  --wait

# Inspect
openstack stack show mystack -c stack_status
openstack stack resource list mystack
openstack stack output show mystack vm_ip
openstack stack event list mystack

# Update a running stack
openstack stack update mystack -t /tmp/basic-stack.yaml --wait

# Delete (removes ALL resources)
openstack stack delete mystack --wait
Task 13.2 — Exam Style: Template with Security Group + Floating IP
Write a Heat template that creates an instance with a security group (SSH + ICMP) and a floating IP. Output the floating IP address.
# File: /tmp/fip-stack.yaml
heat_template_version: 2021-04-16
description: Instance with floating IP

parameters:
  public_net:
    type: string
    default: public
  private_net:
    type: string
    default: app-net

resources:

  web_secgroup:
    type: OS::Neutron::SecurityGroup
    properties:
      name: heat-sg
      rules:
        - protocol: icmp
        - protocol: tcp
          port_range_min: 22
          port_range_max: 22
        - protocol: tcp
          port_range_min: 80
          port_range_max: 80

  web_server:
    type: OS::Nova::Server
    properties:
      name: heat-web
      image: ubuntu-jammy
      flavor: app.small
      networks:
        - network: { get_param: private_net }
      security_groups:
        - { get_resource: web_secgroup }

  web_fip:
    type: OS::Neutron::FloatingIP
    properties:
      floating_network: { get_param: public_net }
      port_id: { get_attr: [web_server, addresses, app-net, 0, port] }

outputs:
  public_ip:
    description: Floating IP
    value: { get_attr: [web_fip, floating_ip_address] }
  server_id:
    description: Server UUID
    value: { get_resource: web_server }
