The certification
What is the COA?
Certified OpenStack Administrator

The COA is the only professional certification issued by the Open Infrastructure Foundation — the same organisation that governs OpenStack itself. It is a performance-based exam: you get a live OpenStack environment, a terminal, and 3 hours to complete real tasks. No multiple choice. No theory questions. You either do it or you don't.

How it works
Exam Format & Environment
What You See On Exam Day
  1. A browser-based exam interface opens with a split screen: task list on one side, live OpenStack environment access on the other.
  2. You get a terminal connected to a real OpenStack cloud — Nova, Neutron, Glance, Cinder, Swift, Heat all running.
  3. You also get access to the Horizon web dashboard at a given URL.
  4. docs.openstack.org is the only external site you can visit. Your bookmarks load there.
  5. Tasks are shown one at a time or as a list. Each has a specific named resource to create/modify/configure.
  6. When time ends, the auto-grader queries the OpenStack API to check if resources exist with correct properties.
  7. Results arrive by email a few hours later.
How the Auto-Grader Works — Critical
The grader checks resource state via API after exam ends. Understanding this changes everything about how you work.
🤖
Names Are ExactIf the task says create volume data-vol and you name it data_vol — that's a fail. Underscore vs hyphen matters. Read every task description twice before typing.
Grader ChecksExample
Resource name matches exactlyVolume must be named exactly as specified
Resource properties are setImage visibility = public if task said public
Relationships existVolume is attached to the correct instance
Roles are assignedUser has correct role on correct project
Quotas are setProject quota matches the specified numbers
Network topologyRouter has external gateway + correct subnet interface
✅
Always VerifyAfter every task, run openstack TYPE show NAME. 10 seconds of verification prevents losing an entire task's marks. Build this as a habit during all 21 days of practice.

12 Things That Cost Candidates Marks — Avoid All of These
  1. Forgetting resize confirm. After server resize, the instance sits in VERIFY_RESIZE. Must run resize confirm.
  2. Router with no subnet interface. Set the external gateway AND add the subnet interface — two separate commands.
  3. Floating IP allocated but not associated. floating ip create and server add floating ip are separate steps.
  4. Wrong resource name. data-vol ≠ data_vol. Read twice, type once.
  5. Volume in error state when attaching. Always check volume show X -c status = available before attach.
  6. Heat stack YAML indentation error. Always validate: openstack orchestration template validate -t file.yaml
  7. Image deactivated and forgotten. If you deactivate for one task, reactivate when done.
  8. Security group created but not applied. Create the SG first, then reference it in server create.
  9. Swift ACL not verified. Always openstack container show NAME after setting ACL.
  10. Quota set on wrong project. Double-check project name before quota set commands.
  11. Keypair created but not injected. Need --key-name mykey in server create.
  12. Deleting resources that should stay. Never delete anything unless the task specifically asks for it.

Environment Setup
MicroStack — Your Practice Lab
MicroStack is Canonical's snap-based single-node OpenStack. It's your exam simulator. Install it once and use it every day.
# On Ubuntu 22.04 LTS (recommended)
sudo snap install microstack --channel 2024/stable

# Initialise (takes 10-20 minutes)
sudo microstack init --auto --control

# Get admin credentials
sudo snap get microstack config.credentials.keystone-password

# Source admin environment
source /var/snap/microstack/common/etc/microstack.rc

# Verify everything is running
openstack service list
openstack compute service list
openstack network agent list

# Create a test image to use throughout (cirros is small and fast)
wget http://download.cirros-cloud.net/0.6.2/cirros-0.6.2-x86_64-disk.img
openstack image create cirros \
  --disk-format qcow2 \
  --container-format bare \
  --file cirros-0.6.2-x86_64-disk.img \
  --public

# Verify image is available
openstack image list
💡
Alternative: DevStackIf MicroStack install fails, use DevStack: git clone https://git.openstack.org/openstack-dev/devstack && cd devstack && ./stack.sh. DevStack is the community standard practice environment.
Daily Startup Sequence — Run Every Study Session

Keyboard Layout & Browser Setup
🚨
US English Keyboard OnlyThe COA exam uses US English keyboard layout only. If you type on AZERTY (French) or any other layout during practice, your muscle memory will fail you when the exam forces US layout. Switch NOW and practice everything on US layout for all 21 days.

Chrome Setup — Do This Before Exam Day
  1. Install Chrome or Chromium (the only supported browsers for the exam).
  2. Disable ALL browser extensions except the official proctor plugin (installed during exam check-in).
  3. Clear cache and cookies the day before the exam.
  4. Set bookmarks now — see next section. Bookmarks work during the exam.
  5. Test your webcam and microphone in Chrome settings.
  6. Disconnect any second monitor. Proctors require single-screen setup.

The day itself
Exam Day Protocol
72 Hours Before
  • Run an internet speed test — need stable 10Mbps+ (webcam + screen share + Horizon all run simultaneously)
  • Confirm Chrome is installed and bookmarks are set
  • Test webcam and microphone in Chrome
  • Set keyboard to US English layout
  • Locate your government-issued photo ID (passport or national ID — physical, not phone photo)
  • Disconnect second monitor if you have one
Day of Exam — Room Requirements
  • Empty room — you must be the only person. Proctor checks via webcam before start.
  • Clean desk — no papers, notebooks, phones. Water in a clear glass is OK.
  • No headphones or earbuds
  • No TV, music, or background conversation
  • Good lighting — proctor must see your face clearly. Don't sit with window behind you.
  • Be ready for room scan — proctor will ask you to show the whole room with your webcam

The strategy
Scoring Strategy & Task Order
Optimal Task Order — Maximum Marks for Time Spent
#DomainWhy First
1stIdentity (Keystone)Fast CLI. Creates users/projects that other tasks need. Quick marks.
2ndImage (Glance)Upload once. All compute tasks need an image. Do early.
3rdNetworking (Neutron)Network must exist before instances. Foundation for 30% marks.
4thCompute (Nova)Biggest mark pool (35%). Do after network is ready.
5thBlock Storage (Cinder)Attach to running instances. Medium effort, solid 10% marks.
LastHeat + Object StorageHeat takes longest. Swift is only 5%. Do if time allows.
The Verification Habit — Every Single Task
# Run after EVERY task before moving to the next
openstack TYPE show NAME

# Specific verifications
openstack server show NAME -c status -c flavor -c image -c security_groups
openstack router show NAME -c external_gateway_info
openstack volume show NAME -c attachments -c status
openstack stack show NAME -c stack_status
openstack role assignment list --project PROJ --user USER --names

# Validate Heat BEFORE deploying
openstack orchestration template validate -t mystack.yaml
